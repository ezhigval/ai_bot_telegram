## Roadmap проекта

### Этап 1. Базовый Telegram-бот (готово частично)

- ✅ Инициализировать Go-проект и модуль.
- ✅ Настроить `.gitignore` и `.env.example`.
- ✅ Реализовать минимальный Telegram-бот (текстовые сообщения, эхо-ответ).
- ✅ Добавить базовый логгер и конфиги (структура настроек).

### Этап 2. Каркас ИИ-агента

- ✅ Создать пакет `internal/agent`:
  - интерфейс `Agent` (метод `Reply(ctx, input) -> output`);
  - интерфейсы `LLMClient`, `Memory`, `Tool`.
- ✅ Реализовать простую заглушку-агента (без реального LLM), чтобы продемонстрировать поток данных.
- ✅ Привязка агента к Telegram-слою (`internal/telegram`), заготовки под текст, голос, файлы и медиа.

### Этап 3. Память и обучение

- ⬜ Добавить `internal/storage` с локальной базой (например, SQLite или JSONL-файлы).
- ⬜ Хранить историю диалогов и метаданные сообщений.
- ⬜ Добавить простой механизм извлечения контекста из истории (по пользователю).

### Этап 3.1. Персональные профили и кросс-канальная память

- ⬜ Ввести сущность «пользовательский профиль» (единый ID для Mac и Telegram).
- ⬜ Привязать историю диалога к профилю, а не к конкретному каналу.
- ⬜ Добавить базовые пользовательские настройки (тон, стиль, предпочтения по формату ответов).

### Этап 4. Интеграция LLM без финансовых затрат

- ⬜ Исследовать варианты:
  - локальные модели (через `llama.cpp`, `ollama` или аналоги);
  - бесплатные публичные API с разумными лимитами.
- ⬜ Обернуть выбранный вариант в `LLMClient`.
- ⬜ Подключить к ядру агента.

### Этап 5. Инструменты (Tools)

- ⬜ `WebSearchTool`: веб-поиск через бесплатные API.
- ⬜ `FileAnalysisTool`: базовый анализ документов, PDF, таблиц.
- ⬜ `MediaTool`: обработка изображений, видео, голосовых.
- ⬜ Маршрутизация: выбор инструмента в зависимости от типа входного сообщения.

### Этап 6. Голосовые сообщения и ответы голосом

- ⬜ Поддержка получения и распознавания голосовых сообщений.
- ⬜ Поддержка генерации голосового ответа (если Telegram API для ботов позволяет отправку голосовых/voice note).
- ⬜ Настройки предпочтения формата ответа (текст/голос).

### Этап 7. Mac-клиент

- ⬜ Спроектировать архитектуру macOS‑клиента (Swift/SwiftUI).
- ⬜ Определить протокол общения с ядром (HTTP/WebSocket API).
- ⬜ Реализовать базовый UI чата с поддержкой текста.
- ⬜ Добавить поддержку отправки файлов и медиа в ядро.

### Этап 8. Деплой на бесплатный сервер

- ⬜ Подобрать хостинг/сервер с достаточными ресурсами и бесплатным тарифом.
- ⬜ Подготовить Dockerfile и инструкции по деплою.
- ⬜ Настроить логирование и мониторинг.
